# Verify exits non-zero on mutation damage

`gunk verify` is the one command whose exit code reflects a failure state: it exits non-zero when it detects damage attributable to a mutation — a dangling reference to a just-trapped path, or a user-configured verify command failing. This does not contradict ADR-0004. That decision bans *findings* as failure: gunk in the repo never fails a run, because judging content is advisory. Verify is not judging content — it answers "did the mutation Gunk Buster just performed break anything?", a pass/fail question about the tool's own action that scripts and MVP 4's MCP tools need in the exit code. Pre-existing breakage (broken links that predate the mutation) is reported as informational context and never fails verify. On failure, human output ends with the exact `gunk restore` command(s) that undo the mutation.
