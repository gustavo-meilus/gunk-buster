# ECHO requires substantial content overlap

Title and heading similarity may nominate documents for comparison, but ECHO evidence requires at least 80% containment of the smaller document in the larger and at least three matching substantive blocks. Paragraphs, individual list items, table rows, and fenced code blocks are blocks; headings, blank blocks, and blocks shorter than 40 normalized characters are excluded. Markdown presentation syntax and prose case are normalized, while code content remains exact apart from line endings. Added material in the larger copy therefore does not disguise duplication, while tiny documents sharing boilerplate do not qualify. The comparison uses this fixed product threshold and deterministic normalized blocks rather than repository-specific tuning, embeddings, or an LLM. We chose this over the original heading-only comparator because shared document structure is common and cannot by itself demonstrate duplicated content.
